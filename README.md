# GPU Facial Recognition Prototype

A Python-based facial recognition prototype built around OpenCV, `face_recognition`, dlib, CUDA, and cuDNN. The project provides a complete capture-train-recognize workflow while keeping each stage isolated in a dedicated script.

The repository contains source code and documentation only. Runtime artifacts such as captured images, trained encodings, logs, and virtual environments are generated by each installation and excluded from version control.

## Architecture

| Script | Purpose |
| --- | --- |
| `01_collect_data.py` | Opens a webcam and captures labeled training images. |
| `02_train_gpu.py` | Detects faces with dlib's CNN detector and serializes 128-dimensional face encodings. |
| `03_recognize.py` | Runs live webcam recognition with status overlays, FPS, CUDA diagnostics, and match distances. |

The recognition script includes a generic protected-action placeholder that can be replaced by application-specific behavior.

## Requirements

- Python 3.10 or newer.
- OpenCV-compatible webcam.
- NVIDIA GPU and current NVIDIA driver for GPU acceleration.
- CUDA Toolkit available through `CUDA_PATH` or `PATH`.
- cuDNN installed through the NVIDIA installer or the `nvidia-cudnn-cu13` Python package.
- CMake and a C++ compiler when building dlib from source.

On Windows, Visual Studio Build Tools with the C++ workload is recommended.

## Installation

Clone the repository, enter the project directory, and create a virtual environment:

```powershell
git clone <repository-url>
cd gpu-facial-recognition-prototype
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Upgrade packaging tools:

```powershell
python -m pip install --upgrade pip "setuptools<81" wheel
```

Install dependencies:

```powershell
python -m pip install -r requirements.txt
```

On Windows, rebuild dlib with CUDA and cuDNN support:

```powershell
.\scripts\build_dlib_cuda_windows.ps1
```

The normal PyPI install can produce a CPU-only dlib build. The build script installs the cuDNN wheel, generates the MSVC import library required for cuDNN, rebuilds dlib from source, and verifies CUDA support at the end.

If PowerShell blocks script execution, run it for the current process with:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_dlib_cuda_windows.ps1
```

For a specific NVIDIA architecture, pass `-CudaArchitectures`. For example, RTX 40-series GPUs use compute capability `89`:

```powershell
.\scripts\build_dlib_cuda_windows.ps1 -CudaArchitectures 89
```

## GPU Verification

Verify that dlib can see CUDA:

```powershell
python -c "import dlib; print('DLIB_USE_CUDA:', dlib.DLIB_USE_CUDA); print('CUDA devices:', dlib.cuda.get_num_devices() if dlib.DLIB_USE_CUDA else 0)"
```

Expected result:

```text
DLIB_USE_CUDA: True
CUDA devices: 1
```

If `DLIB_USE_CUDA` is `False`, dlib was not compiled with CUDA and cuDNN support. Rebuild dlib after confirming CUDA Toolkit, cuDNN, CMake, and the C++ compiler are correctly installed.

On Windows, rerun:

```powershell
.\scripts\build_dlib_cuda_windows.ps1
```

## Workflow

### 1. Collect Training Images

Capture images for one identity:

```powershell
python .\01_collect_data.py --name person_name
```

Controls:

- `c` captures the current webcam frame.
- `q`, `Q`, `Esc`, or closing the window exits.

Dataset layout:

```text
dataset/
  person_name/
    person_name_YYYYMMDD_HHMMSS_microseconds_0001.jpg
```

For best results:

- Use one visible face per image.
- Capture multiple angles and lighting conditions.
- Include the expected recognition distance.
- Avoid motion blur and heavy backlighting.
- Keep the face large enough for the CNN detector to localize it.

### 2. Train Encodings

Train with the default CNN detector settings:

```powershell
python .\02_train_gpu.py
```

Quiet mode:

```powershell
python .\02_train_gpu.py --quiet
```

Useful options:

```powershell
python .\02_train_gpu.py --upsample 1 --jitter 1 --multi-face-policy skip
```

The trainer reports:

- total images per identity
- CUDA and dlib status
- detected face counts
- skipped images
- multi-face images
- per-identity encoding totals
- processing time and throughput

By default, images with multiple detected faces are skipped. This avoids accidentally assigning several faces to the same identity.

### 3. Run Live Recognition

Start the recognizer:

```powershell
python .\03_recognize.py --debug
```

Useful options:

```powershell
python .\03_recognize.py --scale 0.5 --upsample 1 --tolerance 0.52 --debug
```

Controls:

- `q`, `Q`, `Esc`, or closing the window exits.

The recognition overlay shows:

- current recognition status
- detected face count
- FPS
- CUDA device count
- scale, tolerance, and upsample settings
- match distance for unknown faces

## Tuning

Use these settings when detection or recognition needs adjustment:

| Option | Effect |
| --- | --- |
| `--scale` | Higher values preserve more detail and help distant faces, but reduce FPS. |
| `--upsample` | Higher values help detect smaller faces, but increase GPU workload. |
| `--tolerance` | Higher values accept looser matches; lower values are stricter. |
| `--debug` | Prints per-frame detection and matching details. |

Recommended starting points:

```powershell
python .\03_recognize.py --scale 0.5 --upsample 1 --tolerance 0.52 --debug
```

For longer-distance detection:

```powershell
python .\03_recognize.py --scale 0.75 --upsample 1 --tolerance 0.52 --debug
```

For stricter matching:

```powershell
python .\03_recognize.py --scale 0.5 --upsample 1 --tolerance 0.45 --debug
```

## Troubleshooting

### CUDA is not enabled

Run:

```powershell
python -c "import dlib; print(dlib.DLIB_USE_CUDA)"
```

If the result is `False`, rebuild dlib with CUDA and cuDNN support. Confirm that `nvcc`, CMake, and the C++ compiler are available in the same terminal environment used for installation.

### No face detected

Try:

```powershell
python .\03_recognize.py --scale 0.75 --upsample 1 --debug
```

Also collect clearer images at the target distance and retrain.

### Face detected but not recognized

Try a slightly higher tolerance:

```powershell
python .\03_recognize.py --tolerance 0.55 --debug
```

If matches become too loose, lower the tolerance again and improve the dataset.

### Many images are skipped during training

The trainer skips images with multiple faces by default. Capture training images with only one visible face, or intentionally override the behavior:

```powershell
python .\02_train_gpu.py --multi-face-policy encode-all
```

Use `encode-all` only when every detected face in the image belongs to the same identity.
